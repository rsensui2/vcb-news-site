概要

Claude Code on Amazon Bedrockの設定完了後、「お、使いこなせてきたぞ！」と実感できるようになるためのポイント10個を紹介。

1. セットアップの最適化

Shift+Enter による改行入力

/terminal-setup

長いプロンプトを書く際の操作性が向上。

通知をONにする

長時間の自律作業が終わったら通知で連絡をもらうように設定。

音声入力を活用する

タイピングの3〜4倍の情報量を、背景や意図を含めて自然に伝えられる。

2. CLAUDE.md を作成し、育てる

CLAUDE.mdとは？

Claudeにプロジェクトの背景知識（コンテキスト）を持たせるための設定ファイル。

作り方

/init

コードベースを分析し、プロジェクトに合わせたCLAUDE.mdのひな形を自動生成。

設定例

# Project Context
When working with this codebase, prioritize readability over cleverness.

## Key Directories
- `app/models/` - database models
- `app/api/` - route handlers

## Standards
- Type hints required on all functions
- pytest for testing

3. 便利な機能とコマンド

起動コマンド

| コマンド | 内容 |

|---------|------|

| claude | 対話セッションを開始 |

| claude --continue | 直近のセッションを継続 |

| claude --resume | 過去のセッションから選択して再開 |

| claude --dangerously-skip-permissions | YOLOモード（全承認スキップ） |

Escキーで中断

  • Esc 1回: 即座に停止
  • Esc 2回: 前のメッセージを編集（/rewind）

!（Bashコマンド実行）

!git status や !ls で直接実行。

@ファイル選択

@ でプロジェクト内のファイル一覧を表示。

管理コマンド

| コマンド | 内容 |

|---------|------|

| /clear | 新しい会話を開始 |

| /compact | 会話を要約してコンテキスト圧縮 |

| /rewind | 会話を過去に戻す |

| /usage | 使用量を確認 |

4. settings.json で権限を制御する

{
  "permissions": {
    "allow": [
      "Bash(npm run lint)",
      "Bash(npm run test:*)"
    ],
    "deny": [
      "Bash(curl:*)",
      "Read(./.env)"
    ]
  }
}

5. 一般的なワークフロー

Plan → Confirm → Code → Commit

〇〇の根本原因を特定し、いくつか修正案を提案してください。
あなたがコードを書く前に、どのアプローチで行くか私に選ばせてください。

Write Tests → Commit → Code → Iterate → Commit（TDD）

@utils/markdown.java のテストを書いてください。
その後コミットしてください。
そして、テストが通るようにコードを更新してください。

Write Code → Screenshot Result → Iterate

[mock.png] を実装してください。
その後、スクリーンショットを撮り、モック画像と同じ見た目になるまで修正を繰り返してください。

6. プランモードの活用

Shift+Tab で有効化。読み取り専用でコードベースを分析し、実行計画を作成。

7. 拡張思考（Extended Thinking）の活用

Tab を押すと有効化。複雑なタスクに対してより多くの計算時間とリソースを割り当て。

向いているタスク:

  • 複雑なアーキテクチャの変更を計画
  • 複雑な問題のデバッグ
  • 新機能の実装計画
  • 異なるアプローチ間のトレードオフを評価

8. カスタムスラッシュコマンドの活用

.claude/commands/ ディレクトリにmarkdownファイルを配置。

例: .claude/commands/create-commit.md

直近のコードの差分を確認したうえで、コミットメッセージを作成してください。
ルールは rules/commit-message.md を参照してください。

/create-commit で呼び出し可能。

9. MCPサーバーによる機能拡張

Tavily（ウェブ検索）

claude mcp add -e TAVILY_API_KEY=your_token -- tavily-mcp npx -y tavily-mcp@0.1.3

Context7（最新ドキュメント取得）

claude mcp add context7 -- npx -y @upstash/context7-mcp

Playwright（ブラウザ操作自動化）

claude mcp add playwright npx @playwright/mcp@latest

AWS MCPサーバー

  • aws-api-mcp-server: AWSリソースの作成・管理
  • aws-knowledge-mcp-server: AWS公式ドキュメントにアクセス
  • aws-pricing-mcp-server: 料金情報の取得

10. その他 Tips

GitHub連携

GitHub CLI (gh) をインストールすると、Issue/PRの操作が可能。

「Issue #123 を修正して PR を作成して」のような一連の作業を任せられる。

画像の活用

  • UIの問題をスクリーンショットで伝える
  • デザインモックを渡して実装させる
  • 実装結果をスクリーンショットで確認

困ったらClaudeに聞く

Claude Codeは自分のドキュメントをよく知っている。
