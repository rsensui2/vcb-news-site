事前準備・想定環境

PC（ローカルマシン） — この記事の コマンド例は macOS のターミナル を想定しています。Windows / Linux でも Ollama と OpenClaw が動く環境 なら、考え方は同じです（パスやショートカットだけ自分の環境に読み替えてください）。

OpenClaw 導入済み

OpenClaw の gateway が動いている

導入の全体フロー

Ollama が入っているか確認する

Gemma 4 を pull する

openclaw onboard でセットアップ（質問形式の画面）を開く

Existing config detected → Update values で Ollama を追加

チャネル・スキルは壊さない のを優先して最小限で通す

Gateway を Restart

デフォルトが ollama/gemma4:e4b になったか確認する

実際の手順

Step 1. いまの OpenClaw の状態を確認する

最初にOpenClawを動かしているパソコンのローカルのターミナルで現状を見ます。

openclaw config get agents.defaults.model.primary

openclaw models list

openclaw gateway status

見たいのはこの3つです。

いまの デフォルトモデル

models list に何が表示されているか

Gateway が動いているか

Step 2. 次にOllama が入っているか確認する

ollama --version

入ってなければ（公式の手順どおり）

例えば：

curl -fsSL https://ollama.com/install.sh | sh

それかここから直接ダウンロードしちゃいましょう！

https://ollama.com/download

無理ならmacの人はここ参照👇

https://docs.ollama.com/macos

Windowsの人はここ👇

https://docs.ollama.com/windows

Step 3. その後ローカルのターミナルでGemma 4 を pull する

まだなら：

ollama pull gemma4:e4b

確認：

ollama list

ここに gemma4:e4b が出ていれば OK です。

Step 4. openclaw onboard を実行する

openclaw onboard

ここから ぽちぽちボタン押すフェーズ画面です。ここで止まる人多いので、ここを下で詳しく解説しますね👇

こんなの☝️

一番つまずきやすいのはこの先なので、選択肢ごとに書きます

その前に先に理解しておくこと

1.openclaw onboard は「初回専用」ではない

openclaw onboard を実行すると、ターミナルに 英語の質問が順に出て、矢印キーや Enter で選びながら設定を進める画面 になります。IT の説明では wizard（ウィザード） と呼ばれることが多いですが、「質問に答えるセットアップ」と考えてください。

実行した直後は、よく次の不安になりがちです。

「え、今のOpenClawの設定全部消して最初から？」

でも実際は、既存設定があると検出されて、どうするか聞かれる 仕様です。

つまり onboard は、初回だけではなく「設定の更新・追加のための質問形式のセットアップ」 として何度でも使えます。

  1. あと「Gemma4を使えるようにする」と「デフォルトにする」は別

Gemma 4 を OpenClaw で使えるようにする ことと、

毎回の既定を Gemma 4 にする ことは別です。

最終的に

openclaw config get agents.defaults.model.primary

で

ollama/gemma4:e4b

が返ってきたら、デフォルトが Gemma 4 になっています。

一方 openclaw models list に

ollama/gemma4:e4b

（いままで使っていたモデル、例：openrouter/moonshotai/kimi-k2.5 や openrouter/anthropic/claude-sonnet-4.6 など）

のように並んでいれば、他のモデルも残ったまま選べる という意味です。

では設定続けていきましょう👇

openclaw onboardでのセットアップ画面での選び方

4-1. Security warning

I understand this is personal-by-default... のあと Continue? が出ます。

選ぶ: Yes

理由: 今回は個人利用前提で、まずローカル検証から。

4-2. Setup mode

QuickStart と Manual が出ます。

選ぶ: QuickStart

理由: 今回の目的は「Gemma 4 を使えるようにする」ことで、ポートや細部を全部いじることではないから。

4-3. Existing config detected（超重要）

既存があると例のように workspace / model / gateway が表示されます。

そのあとだいたい：

Use existing values

Update values

Reset

が出ます。

選ぶ: Update values

理由:

Use existing values → 今の Kimi 中心のまま、Ollama が足りない

Reset → 危ない（設定をいったん消す方向に進む）

Update values → Ollama / Gemma を足せる

ここを間違えると、「結局 Gemma が増えていない」まま終わりやすいです。

4-4. Model / auth provider

選ぶ: Ollama

理由: ローカル Gemma 4 だから。

4-5. Ollama base URL

だいたいこれで固定で OK です。

http://127.0.0.1:11434

ローカルの Ollama は、通常このアドレスで待ち受けています。

4-6. Ollama mode

Cloud + Local と Local が出ます。

選ぶ: Local

理由:

無料

ローカル

API 不要

外に送りたくない

4-7. Default model

候補に ollama/gemma4:e4b が出ます。

選ぶ: ollama/gemma4:e4b

理由: まず現実的な一本としてこれが扱いやすい。

チャネル設定で詰まらない考え方

5-1. Channel status は「いま使える入口の一覧」

例：

Telegram: configured

Discord: configured

LINE: configured

これは もうそのチャネルは使える という意味です。

今回の目的が 「Gemma 4 を足す」 だけなら、新しいチャネルを増やす必要は基本ありません。

5-2. Select channel で余計なものを選ばない

一覧に Discord / Slack / LINE / Zalo などが並び、カーソルが今回関係ない項目に乗っている こともあります。

方針:

configured のものを壊したくない → 既存は Skip 寄りでいく

新規で要らないチャネルは触らない

初期カーソルに引っ張られない

5-3. Telegram / Discord / LINE は既存運用をそのまま活かす

今回、手元で通した設定では Telegram / Discord / LINE は configured でした。

Gemma 4 だけ足したいなら、チャネルを増やすのではなく、今つながっているボットに新しいモデルを使わせる のが正解に近いです。

Web Search Provider はどうするか

Brave Search / DuckDuckGo / Exa / Skip for now などが出ます。

選ぶ: Skip for now

理由: 今回の本題は Gemma 4。検索は別機能。あとからでいい。

Skills 設定はどうするか

Configure skills now? (recommended) が出ます。

Yes でもよい ですが、依存は 最小限 が基本です。

6-1. Install missing skill dependencies（ここでハマる）

Skip for now / bear-notes / blogwatcher … などがチェック式で並びます。

Enter だけでは進まない ことがあります。Space で最低1つチェック が必要です。

いちばん安全: Skip for now に Space を入れてから Enter

理由: 今は Gemma を使えるようにする のが目的で、skill 依存を大量に入れる段階ではないから。

6-2. Space / Enter が効かないとき（地味に多い）

セットアップの チェック一覧の画面で、

Space 押したつもりがチェックが入らない

Enter しても Please select at least one option

といった状態になることがあります。

この時は ここのボタンをABCにしましょう。反応するようになります！

Hooks はどうするか

Skip for now / boot-md / command-logger … など。

選ぶ: Skip for now

理由: Hooks は便利だけど 本体導入と別。まず 動く が最優先。

Gateway service already installed のところ

Restart / Reinstall / Skip が出ます。

選ぶ: Restart

理由: いま openclaw.json に新設定が反映された ので、動いている gateway に 読み直させる のが先。再インストールは不要なことが多い。

Hatch の画面はどうするか

Hatch in TUI / Open the Web UI / Do this later など。

選ぶ: Do this later

理由: まず セットアップを安全に終わらせて、コマンドで 反映確認 した方が事故が少ない。

導入後の確認コマンド（必ず）

openclaw config get agents.defaults.model.primary

openclaw models list

理想:

primary が ollama/gemma4:e4b

models list に ollama/gemma4:e4b と、もともと使っていたモデル（例：僕なら openrouter/moonshotai/kimi-k2.5、あなたなら openclaw models list に表示されている別名）が 両方並んでいる

今回の最終確認でも、僕の環境ではこの形になりました。

ここまでの到達状態（イメージ）

Ollama mode = Local

Default model = ollama/gemma4:e4b

Telegram / Discord / LINE = configured のまま

つまり 「Gemma 4 をデフォルトにしつつ、既存チャネルも残した」 状態です。

Telegram で「モデルが変わっていないように見える」とき

原因1: デフォルトは変わったが、セッションが古い

/models や /model を触っても、思ったモデルに見えないことがあります。

対処: 新セッションを明示的に切る。

/new ollama/gemma4:e4b

✅ New session started · model: ollama/gemma4:e4b のように返ってくれば、切り替え自体は成功 しています。

その直後に短文で話しかけるとわかりやすいです。

おはよう。あなたのモデルは今何？

原因2: 同じチャネルでも、会話の文脈を引きずっている

対処: 上の /new のあと、短い発話で確認。

原因3: Gemma 4 が重くて、応答が返ってこないように見える

モデル切替は成功 なのに、推論が遅い / 止まって見える ことがあります。

運用のおすすめ使い分け（正直な話）

Gemma 4（e4b）は優秀だし使えるけど、OpenClaw のメインの思考エンジンとして完全にずっと常に固定するかどうかは話別です。

自分はちゃんと何でもかんでも最高！使える！マジでやばい😳

って不誠実な発信するタイプではないので、ここも書いておきます。

結論、用途で切り分けた方がいいです。

現実的には、だいたいこう割るとバランスが取りやすいです。

普段は Gemma で試す。重いときだけ、元の本命モデルに戻す。 が一番現実的です。

一応ターミナルでモデルを切り戻すコマンド

まず必ずこれを打つ：

openclaw models list

出てきた 正確なモデル ID をコピーして、次の set に使います。

例1：いちの環境で Kimi 2.5 に戻すとき

openclaw models set openrouter/moonshotai/kimi-k2.5

例2：Claude（Sonnet など）に戻すときのイメージ

openclaw models set openrouter/anthropic/claude-sonnet-4.6

※ claude-sonnet-4.6 の部分はあなたの models list の表記に合わせてください。 バージョン名が違っていても普通です。

Gemma 4 に再び戻すとき

openclaw models set ollama/gemma4:e4b

この記事を出すタイミングでは、開催からおおよそ 1 週間前に行ったばかりの、いちばん新しいアーカイブになります😆
