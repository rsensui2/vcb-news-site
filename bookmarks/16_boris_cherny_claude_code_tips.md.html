Claude Code Tips from the Creator (Boris Cherny)

№250 / 307 SRCx.com/post ↗ GHGitHub ↗
250BOOKMARK / 307

概要

Claude Code開発者Boris Chernyが、Claude Codeチームから直接集めたTipsを共有。「正しい使い方」は一つではなく、セットアップは人それぞれ。実験して自分に合うものを見つけることが大事。

Tip 1: Do more in parallel（並列作業）

3–5個のgit worktreesを同時に立ち上げ、それぞれで独立したClaudeセッションを並列実行する。

これはチームからの最大の生産性向上Tip。

個人的にはmultiple git checkoutsを使うが、Claude Codeチームの多くはworktreesを好む。

Tip 2: Start every complex task in plan mode（プランモードから始める）

複雑なタスクは必ずプランモードから開始。プランに全力を注げば、Claudeが実装を1-shotで完遂できる。

チームの使い方：

  • 一人のClaudeにプランを書かせ、別のClaudeにスタッフエンジニアとしてレビューさせる
  • 何かがおかしくなった瞬間、プランモードに戻る

Tip 3: Invest in your CLAUDE.md（CLAUDE.mdに投資する）

修正するたびに最後にこう言う：「CLAUDE.mdを更新して、同じミスを二度としないようにして。」

Claudeは自分自身のルールを書くのが驚くほど上手い。

CLAUDE.mdは時間をかけて ruthlessly edit（情け容赦なく編集）し続ける。イテレーションを止めない。

Tip 4: Create your own skills（自分のスキルを作る）

スキルを作ってgitにコミット。全プロジェクトで再利用。

チームからのTips：

  • 1日に1回以上やることはスキルまたはコマンドにする
  • /techdebt スラッシュコマンドを作り、セッション終了時に実行して重複コードを見つけて削除

参考: https://code.claude.com (Extend Claude with skills - Claude Code Docs)

Tip 5: Claude fixes most bugs by itself（バグは大体Claudeが直す）

Slack MCPを有効にして、Slackのバグスレッドを貼り付けて「fix」と言うだけ。

コンテキストスイッチング不要。

または「Go fix the failing CI tests」と言うだけ。howは指示しない。

Claudeをdocker logsに向けるのも有効。

Tip 6: Level up your prompting（プロンプトをレベルアップ）

a. Claudeにチャレンジする

  • 「この変更について厳しく質問して、テストに合格するまでPRを作らないで」
  • Claudeをレビュアーにする
  • 「これが動くことを証明して」と言い、mainとfeatureブランチの動作の差分を取らせる

b. mediocreな結果の後

  • より良い結果を求めて再度プロンプト

Tip 7: Terminal & Environment Setup（ターミナル環境設定）

チームはGhosttyを愛用！

理由：

  • synchronized rendering
  • 24-bit color
  • proper unicode support

Claude管理を楽にするTips：

  • /statusline でステータスバーをカスタマイズ
  • context usage（コンテキスト使用量）と current git branch を常に表示

Tip 8: Use subagents（サブエージェントを使う）

a. リクエストに「use subagents」を追加すれば、Claudeが問題により多くの計算リソースを投入

b. 個別タスクをサブエージェントにオフロードし、メインエージェントのコンテキストウィンドウをクリーンで集中した状態に保つ

c. permission requestsをhook経由でOpus 4.5にルーティング — 判断を任せる

まとめ

  • 並列作業で生産性を最大化
  • プランモードで1-shot実装を実現
  • CLAUDE.mdでClaudeに自己改善させる
  • スキル化で再利用性を高める
  • バグ修正はClaudeに任せる
  • チャレンジングなプロンプトで品質向上
  • Ghostty + statuslineで環境最適化
  • サブエージェントでスケールアウト
