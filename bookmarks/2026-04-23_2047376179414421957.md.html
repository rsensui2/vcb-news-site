概要

Matt Shumer による GPT-5.5 の詳細レビュー X Article。結論: GPT-5.5 は明確なアップグレードだが、従来モデルが既に「異常な水準」に達していたため、日常的なエンジニアリングではアップグレードを実感しにくい。一方で、デザイン・ネイティブアプリ・セキュリティ・マルチエージェントなど、GPTが弱かった領域を埋める「レンジ拡張」が最大の変化。セキュリティ監査では Opus が見逃した脆弱性を発見した実例もある。

まとめ

  • エンジニアリング: 既に「異常に良い」レベルだった GPT-5.3 Codex / 5.4 を上回るが、日常タスクでは体感差が小さい
  • デザイン: Figma や既存UIからの拡張に強い。GPT Image 2 のモックと組み合わせると Opus 並みの出力
  • ネイティブ/モバイル: Swift や Mac アプリ開発で一段と安定
  • セキュリティ: Opus や過去GPTが見逃した実脆弱性を検出。汎用モデル最強クラス（専用チューニングの GPT-5.4-Cyber には劣るが大幅に近づいた）
  • Opus化: トークン効率が上がり低reasoningモードでも実用品質。手触りが「速くて楽しい」
  • Pro: 速いが判断ミスや書き味の劣化（1文1改行のクセ）があり、執筆用途はレグレッション
  • ハーネスが次の鍵: 単体モデル性能は飽和気味、マルチエージェント・オーケストレーションが次の伸びしろ

本文（日本語訳）

TL;DR

  • GPT-5.5 は真のアップグレードだが、日常的なエンジニアリング作業ではその差を感じにくい場合がある。従来モデルが既に異常なほど優秀だったためだ
  • 本格的なソフトウェア作業では例外的に強い。思慮深く、慎重で、私と同じ判断を多く下し、目的が達成されるまで粘り強くイテレートする
  • 最大の物語は「レンジ」。GPTラインの弱点（既存コンテキストからのデザイン、iOS/Macネイティブアプリ、セキュリティなど）を埋めた
  • デザインは大幅に改善。特に既存プロダクトの拡張や Figma からの実装に強い。GPT Image 2 が UI モック生成に優秀なので、組み合わせるとデザイン面の弱さが補完される
  • セキュリティが最も興味深い改善。私のテストでは GPT-5.5 が従来 GPT や Opus が見逃した脆弱性を発見した。全員が自コードベースに対して実行すべき
  • モデルが信頼できるレベルになると、ボトルネックは周辺のハーネスに移る。真のパワーは複数エージェントが連携して大きな目標に向かうシステムで現れる
  • 手触りが Opus 化した。速く、トークン効率が良く、楽しい。低reasoningモードが妥協に感じなくなった
  • Pro はより複雑。素晴らしい瞬間もあるが、私の体験は最良ではなかった。Pro ではトークンを多く使ってほしいのに、逆に効率化されすぎる。執筆も大きなレグレッションだ

フロンティアは奇妙になっている

エンジニアリングは奇妙な段階に入った。最良のモデルは既に例外的なエンジニアだ。コードベースを読み、アーキテクチャを理解し、良い判断を下し、コードを書き、テストし、問題を修正し、動くまで続けられる。これはかつてSFだったが今や当たり前だ。だから GPT-5.5 が良くなっても、日常ワークフローでの劇的変化には結びつきにくい。天井が高くなりすぎて、普通の作業ではモデルに負荷がかからないのだ。

ハーネスが重要になってきた

モデルが信頼できるレベルに達した今、周辺のハーネスが律速になる。複数エージェントが仕事を分担し、互いをクリティークし、専門化し、上位エージェントがチームを率いるCEOのように調整する、そういったマルチエージェント構成が次の大きな解錠要素だと見ている。

セキュリティが最大の新しい理由

GPT-5.5 で最も驚いたのがセキュリティ。従来の GPT や Opus が見つけられなかった実脆弱性を検出した。理論上のナンセンスではなく「これは本当に問題だ」と思える実際の issue だ。タスクを具体的に指定（認証境界のトレース、権限前提の検査、インジェクションリスク、依存関係使用のレビューなど）すると非常に強力に動く。

Opus 化した手触り

Claude Opus / Claude Code は常に「手触りが速い」感覚があり、全体性能で優れていなくても楽しかった。GPT-5.5 はそのギャップを縮めた。トークン効率が上がり、低reasoningでも強い結果が出る。高reasoningは本当に大事な時に温存するが、日常的には低〜中で十分楽しめる。

Pro は速いが複雑

GPT-5.5 Pro は速く、時に素晴らしいが、従来 Pro ほど徹底性を感じない。トークン効率化された結果、Pro に期待する「オーバーシンキング」が薄れた。執筆は 1文1改行のクセがありレグレッション。私宛ではない文書に「なぜそう書くかの舞台裏の推論」を含めてしまうような判断ミスもあった。

いつ何を使うか（著者の現在の使い分け）

  • GPT-5.5: エンジニアリング、ロジック重視、セキュリティ重視、既存システムへの強い実装。バックエンド、アーキテクチャ、モバイル/ネイティブ、統合、コードレビュー、脆弱性ハント
  • Opus: 画像リファレンスなしで UI を生成・拡張する用途、小さなUI変更の即応
  • Gemini: GPT Image 2 → GPT-5.5 が詰まった時のバックアップ、スクラッチからのデザイン

最終考察

GPT-5.5 は明確なアップグレードだが、ほとんどの人のほとんどのタスクでは「既存能力が広く・安全に・信頼できるようになった」という性質の改善。だが押し込んで使う領域では明確に強い。99% の時間は不要だが、必要な 1% での差は大きい。
