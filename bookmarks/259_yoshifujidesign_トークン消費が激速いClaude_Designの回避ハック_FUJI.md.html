概要

Claude Design はリサーチプレビュー中のため独自消費率に加えデザインシステム生成のワークロードが重く、ちょっとした修正でもトークンを急速に消費する。著者FUJI（プレゼンテーションデザイナー）のハック: ①Claude Designでデザインシステム+初期データを作る → ②ローカルにダウンロード → ③修正は Claude Code でローカル編集。Claude Designは「デザインシステム構築 + 初回生成」だけに絞る運用が有効という知見。

まとめ

  • Claude Designはリサーチプレビュー中で独自のトークン消費率が適用される
  • デザインシステム構築のワークロードが重く微修正でもトークンを食う
  • 回避策: Claude Design→ローカルDL→Claude Codeで修正 の 3 段運用
  • Claude Designが活きるのは「デザインシステム + 初回大物生成」、微調整はCode で十分

タイトル: トークン消費が激速いClaude Designの回避ハック

Claude Designはトークンの消費が本当に速い。

こういう投稿をして聞いてみたところ、複数の方から同じように消費が速いというコメントをいただきました。

この原因なのですが、そもそもリサーチプレビュー（試験提供／実験公開）中なのでClaude Design独自のトークンの消費率が適用されているのと、デザインシステムの構築やそれをもとにしたデザインの生成という作業のワークロードがそれなりに重いので、ちょっと修正を加えるだけでもがっつりトークンが消費されてしまうようです。

自分でいろいろ試してみていて、ひとまずの運用でこの速すぎるトークン消費を多少でも回避する方法をご紹介しようと思います。

といってもすごく単純で、

① Claude Designで、デザインシステムと最初のデータを作る

② できたデータセットをローカルにダウンロードする

③ 修正作業はClaude codeでローカルデータをいじる

というふうに、切り分けて使うだけです。

Claude Designは、これでしかできないデザインシステムを使うフェーズだけにできるだけ絞って、なんでもかんでもClaude Designでやらないようにして修正や微調整をClaude codeにまわすとトークン消費を抑えることができます。

生成物を比べてみても、Claude Designが活きるのはデザインシステムの構築と、最初の生成物が大きくて、微調整のレベルでは大きな差はないように感じました。

②と③がポイントで、Claude Designが作ったデータをそのままローカルにダウンロードしておきます。

このデータフォルダで Claude code を起動すれば、各ファイルをそのまま編集してくれるので簡単です。Claude codeに慣れている人ならこちらの方が楽まである。

最初からこうやればよかったんですが、自分はMaxプランを週間制限いっぱいまで使った後でこのやり方に思い至りました。もっと早く気がつけばよかった。。と思ったのでせっかくなので記事にしました。

Claude Designをお使いの方、これから使う方、トークンがあっというまになくなる前に、使いどころを切り分けるやり方を試してみてください。
