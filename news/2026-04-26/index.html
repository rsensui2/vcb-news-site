DeepSeek V4 Preview リリース — 1Mコンテキストを「標準」にしたOSSフラッグシップ

DeepSeek が V4 Preview を OSS 公開。フラッグシップ V4-Pro (1.6T総 / 49Bアクティブ MoE) と高速・省コスト V4-Flash (284B / 13B) の2モデル構成で、全公式サービスで1Mトークンコンテキストを標準化した。Token-wise圧縮 + DSA(Sparse Attention)で長コンテキストの計算/メモリコストを大幅に削減し、APIは即日更新、旧モデルは3ヶ月でサンセット。

キーポイント

  • V4-Pro: 1.6T総 / 49Bアクティブ。Agentic Codingでオープン界SOTA、世界知識はGemini-3.1-Proに次ぐ2位
  • V4-Flash: 284B総 / 13Bアクティブ。V4-Pro並みの推論力で爆速・激安API
  • 全公式サービスで1Mコンテキストがデフォルト化
  • Token-wise圧縮 + DSA、Muon optimizer、On-Policy Distillation、FP4 QAT
  • APIは更新済み、旧モデルは3ヶ月でサンセット
  • Zhihu解析: もはやベンチ競争でなく『効率的long-context推論』という新戦場
クローズド勢(GPT-5.5/Opus 4.7/Gemini 3.1 Pro)に肉薄する性能をOSSで提供し、軸を『長コンテキスト効率』に振ったことで、リポジトリ丸ごと読ませる運用や長文ドキュメント業務の前提が変わる。

Cursor 3.2 — Agents Window大刷新、並列WorktreesとMulti-rootワークスペース

Cursor 3.2 が Agents Window を大幅刷新した。並列Worktreesで異なるブランチをbackgroundで同時に走らせワンクリックでforegroundに昇格、さらにMulti-rootワークスペースで1つのAgentセッションが複数フォルダ(クロスリポ)にまたがる作業を扱えるようになった。

キーポイント

  • Worktrees: 複数ブランチでエージェントを同時並列実行、ワンクリックで切替
  • Multi-root Workspaces: 1セッションが複数フォルダを同時にスコープ
  • GPT-5.5統合 + CursorBench首位の直後に投入された機能強化
  • ダウンロード: cursor.com/download
エージェント並列化(forked-context / parallel worktree)がIDEレイヤーで標準化されつつあり、『同じ依頼を複数ブランチに投げて勝った成果を採用する』運用が現実的に。

GPT-5.5でスライド画像をpptxにほぼ再現するプロンプト — suh_sunaneko

GPT-5.5のマルチモーダル理解 + Codex系コード生成力を使い、スライド画像から編集可能なPPTXを再現するプロンプトを公開。要点は『分解→再現→レンダリング比較→修正』の4ステップを必ず1サイクル以上回すこと。SVG禁止、全体画像化禁止、テキストは編集可能ボックス、装飾は図形、アイコンはPNG/JPEG切り出し。

キーポイント

  • スライド画像→PPTX変換の精度向上には『分解→再現→検証→修正』の4ステップを必ず回す
  • SVG禁止、スライド全体の画像貼り付け禁止
  • テキストは編集可能テキストボックス、装飾は図形、アイコンはPNG/JPEGで切り出し
  • スライドサイズは16:9 (13.333 × 7.5 inch)
  • 4006ブックマーク・54万表示と高エンゲージメント
nanobanana slide generator が『ゼロから生成』なのに対し、本プロンプトは『既存スライド再現』のユースケース。顧客提案の競合スライド再現や社内テンプレ移行に活用可能で、検証ステップを組み込むプロンプト設計は他のAIワークフローにも応用できる。
